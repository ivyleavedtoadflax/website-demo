Contribute

You don't need to be technical to add or improve a page. Here's how a change makes its way onto the site.

How a change is made

  1. Describe what you want. Ask for a new section or an edit — for example, "add a page on stakeholder mapping" or "rewrite the intro to be plainer".
  2. A draft is created. The change is prepared on its own branch, separate from the live site, so nothing changes for visitors yet.
  3. A preview link appears. The draft is published to its own temporary web address so you can see it as a real page, not just as code.
  4. Someone reviews it. A reviewer reads the change, suggests edits, or approves it.
  5. It goes live. Once approved, the draft is merged and the live site updates automatically. The preview link is then retired.

Why it's done this way

  • Nothing breaks the live site by accident. Drafts are separate until someone approves them.
  • Every change is reviewable. You can always see exactly what changed, and who changed it.
  • Mistakes are easy to undo. Because changes are tracked, any edit can be rolled back.
  • Automated checks catch common errors — such as broken links — before a change can go live.
The short version: describe the change, get a preview link, have it reviewed, and it's live. The technical steps are handled for you.

Good first contributions

  • Add a method you use to Methods & tools.
  • Add a worked example or short case study.
  • Improve the plain English on any page.
